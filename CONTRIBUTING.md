# Contributing to EmergencyStorage

Thanks for your interest in improving EmergencyStorage! This project welcomes contributions of documentation, new data sources, reliability fixes, and tooling improvements.

## Getting Started

1. Read the full guide in [docs/CONTRIBUTING.md](docs/CONTRIBUTING.md) for environment setup, coding standards, and workflow details.
2. Make sure scripts are executable: `chmod +x emergency_storage.sh scripts/*.sh`.
3. Prefer small, focused pull requests and include testing notes.

## Development Basics

- Run relevant test scripts from `tests/` (for example `./tests/test_auto_update.sh`) before submitting.
- Use `shellcheck` where available for shell scripts.
- Follow the existing logging and error-handling patterns.

## Ground Rules

- Be respectful and collaborative (see our [Code of Conduct](CODE_OF_CONDUCT.md)).
- Update documentation when behavior changes.
- Link issues in your pull request description when applicable.

If you are unsure where to start, open an issue to discuss ideas first. We appreciate your help keeping offline knowledge available for everyone.
